# 2026-06-02 · 梦境支配者状态标签体系 · 设计文档 

<title>2026-06-02 · 梦境支配者状态标签体系 · 设计文档 </title>

<callout emoji="📌">
**结论：**P0 的状态标签体系不做传统“火水雷冰”属性克制，也不把圣遗物权柄直接变成复杂元素表。当前版本只保留 5 个一级状态标签：**安眠、缓速、击退、标记、护盾**。它们服务的是床防守、三猫切换、局内强化和梦境事件，而不是单独撑起一套重数值系统。
</callout>

# 1. 设计目标

状态标签是连接“角色技能、怪物行为、三选一强化、梦境事件、UI 反馈”的轻量规则层。它要让玩家能快速理解一场战斗里发生了什么：谁在保护床，谁在拖慢怪物，谁把敌人推离床边，谁让目标更适合集火，谁给猫咪或主人提供安全窗口。

这套体系不承担世界观权柄的全部表达。圣遗物里的王权、守护、安眠、音律、时间、结界、远视、召唤、律法、迷宫、净化、镇压等词，仍然保留为角色文化和后续内容扩展用语；进入 P0 战斗规则时，先被翻译成可开发、可显示、可验收的 5 个状态标签。

# 2. P0 一级状态标签

| 标签 | 主要目标 | 战斗意义 | 当前设计来源 | P0 验收口径 |
|-|-|-|-|-|
| **安眠** | 主人 / 队伍 / 床区 | 稳定主人睡眠度，降低“主人快醒”的压力，形成牧师猫和梦境事件的核心识别。 | 铃音 / 安眠巫女、安眠铃、鸟居梦封、天岩户神乐铃。 | 至少能表现为恢复睡眠度、延缓睡眠度下降、降低床边干扰三选一。 |
| **缓速** | 敌人 / 区域 | 拖慢敌人向床推进，让玩家获得走位、切猫和处理猫砂盆/喂食器的时间。 | 奈芙月沙炮台、流沙牢笼、沙尘王权；铃音鸟居梦封；棉花梦毯织网。 | 敌人移速或推进节奏明显降低；持续时间结束后恢复正常。 |
| **击退** | 敌人 / 床边威胁 | 把已经逼近床的敌人推出危险线，是床防守里最直接的救场手段。 | 塞班圆盾冲锋、王冠圣裁；铃音鸟居梦封。 | 普通敌人发生位移；Boss 和重型敌人可以只触发短硬直或击退削弱版。 |
| **标记** | 精英怪 / Boss / 关键目标 | 让玩家知道“现在该集火谁”，也为三猫切换和三选一强化提供组合入口。 | 棉花星线标记 / 织梦印；奈芙攻击受控敌人增伤；圣遗物“破译 / 审判 / 律法”方向。 | 被标记目标在 UI 上可辨识，并至少触发一次增伤、掉落、控制延长或优先索敌效果。 |
| **护盾** | 猫咪 / 床区 / 队伍 | 制造短暂安全窗口，让前排猫能承压，让玩家有时间处理本能数值或切换节奏。 | 塞班银誓护盾；铃音救场技能；圣遗物“守护 / 结界”方向。 | 可吸收伤害或降低伤害；持续期间 UI 清晰显示剩余保护状态。 |

# 3. 派生表现与后续扩展

P0 中不建议把“禁锢、嘲讽、反射、震慑、净化、召唤、时间、结界”全部做成一级标签。它们可以先作为一级标签的派生表现存在：禁锢是缓速的强表现，嘲讽是护盾或守护技能的行为表现，反射和震慑适合玉衡等后续角色，净化适合铃音强化或后期异常系统，召唤、时间、迷宫更适合圣遗物和梦境事件扩展。

| 派生词 | 归属一级标签 | 处理方式 |
|-|-|-|
| 禁锢 | 缓速 | P0 先作为“缓速到 0 或接近 0”的强控表现，不单独开 UI 图标。 |
| 嘲讽 | 护盾 | 用于塞班开盾后的敌人目标重定向，表现为护盾技能的附加行为。 |
| 结界 | 护盾 / 安眠 | 防守语义归护盾，睡眠稳定语义归安眠，避免两套效果混淆。 |
| 反射 / 反制 | 后续扩展 | 适合玉衡与“因果钱”体系，暂不进入 P0 一级标签。 |
| 净化 | 后续扩展 | 等异常状态出现后再加入，否则 P0 没有足够对象承载。 |
| 时间 / 迷宫 / 召唤 | 梦境事件 / 圣遗物 | 适合做局内规则扰动，不应挤占 P0 战斗状态标签。 |

# 4. 标签规则接口

每个状态标签都必须有明确的触发来源、作用对象、持续方式和结束条件。设计上不要只写“施加减速”或“获得护盾”，而要写清楚来源技能、作用范围、持续时长、叠加策略、UI 表现和被覆盖规则。这样后续从文档进入开发时，不会因为词语好看但规则不清而反复返工。

| 字段 | 说明 |
|-|-|
| StatusTagId | 标签 ID，建议使用 sleep_stable、slow、knockback、mark、shield。 |
| DisplayName | 玩家看到的名称：安眠、缓速、击退、标记、护盾。 |
| TargetType | 作用对象：主人、猫咪、敌人、床区、地图区域。 |
| Duration | 即时、短时、区域持续、波次持续四类即可，不要一开始拆太细。 |
| Magnitude | 效果强度，例如缓速百分比、护盾吸收量、安眠回复量、标记增伤值。 |
| StackPolicy | 叠加策略：刷新持续时间、取最高强度、可叠层、不可叠加。P0 默认优先用刷新和取最高，少用叠层。 |
| VisualToken | 颜色/图标/特效关键词，用于 UI 和美术沟通。 |

# 5. 角色状态标签映射

<callout emoji="✅">
**四猫 + 棉花校准结论：**当前 4 只 P0 角色不新增一级状态标签；棉花作为 5 级解锁指挥猫，沿用既有标签而不扩表。塞班锁定“击退 + 护盾”，奈芙锁定“缓速 + 标记”，铃音锁定“安眠 + 护盾”，影丸锁定“标记 + 缓速”，棉花锁定“标记 + 缓速”。嘲讽、禁锢、驱散、隐身、诱敌、影痕、织梦印都只作为技能行为或角色自我资源，不进入 P0 一级状态标签。
</callout>

| 角色 | 战斗职责 | 主标签 | 次标签 | 技能接入口 | 不进入一级标签 |
|-|-|-|-|-|-|
| **塞班 / 圣剑士** | 床边防线、前排重置、护床承压。 | **击退** | **护盾** | 圆盾冲锋、王冠圣裁触发击退；银誓护盾触发护盾；银誓守护强化床边击退和承压能力。 | 嘲讽是护盾附加行为；骑士、王冠、圣剑只做视觉和叙事表达。 |
| **奈芙 / 月沙特工** | 区域压制、清怪、控场增伤。 | **缓速** | **标记** | 月沙炮台提供轻缓速；流沙牢笼表现为强缓速或短禁锢；沙尘王权提供大范围强缓速；王权审判读取受控或被标记目标增伤。 | 禁锢归入缓速的强表现；飞行打落是沙尘王权的技能行为，不新增状态标签。 |
| **铃音 / 安眠巫女** | 睡眠稳定、救场、队伍安全窗口。 | **安眠** | **护盾**  <br/>补充：少量缓速 / 击退 | 安眠铃或鸟居梦封必须接入主人睡眠度稳定或恢复；鸟居梦封可附带击退与缓速；冰华祝祷保留救场回复。 | 治疗和驱散不是当前 5 个一级标签；驱散待后续异常系统明确后再处理。 |
| **影丸 / 逐影者** | 单体爆发、追击、标记收割。 | **标记** | **缓速** | 弧光斩、掠影施加弧光印记；百影追猎锁定标记目标；光斑诱引落地后提供小范围缓速。 | 影痕是自我资源；隐身、诱敌、追击是行为表现，不新增暗影、刺杀或潜行标签。 |
| **棉花 / 梦织女** | 飞毯绕场、路径织网、集火指引。 | **标记** | **缓速** | 星线标记施加标记；梦毯织网和星毯织梦提供区域缓速与路径封锁；织梦印可作为被标记目标的增伤 / 延时结算入口。 | 飞毯、织梦、梦线是视觉和角色机制；织梦印是角色自有机制，不新增梦境、编织或飞毯标签。 |

## 四猫 + 棉花校准验收口径

| 角色 | 必须验收 | 暂不验收 |
|-|-|-|
| **塞班** | 普通敌人可被击退；Boss 仅短硬直或削弱击退；护盾 UI 的吸收量和持续时间清晰。 | 不单独制作嘲讽 UI。 |
| **奈芙** | 缓速强度、范围、持续时间清晰；王权审判可以读取缓速或标记目标形成增伤。 | 不单独制作禁锢 UI。 |
| **铃音** | 至少一个技能直接影响主人睡眠度；护盾或结界不能和生命治疗混淆；鸟居梦封的缓速 / 击退只作为救场补充。 | 不制作完整驱散异常系统。 |
| **影丸** | 弧光印记使用“标记” UI 或轻量变体；大招只锁定标记目标；光斑诱引用“缓速”结算。 | 不制作独立潜行标签或影痕通用状态。 |
| **棉花** | 星线标记使用“标记” UI 或轻量星线变体；梦毯织网 / 星毯织梦用“缓速”结算；织梦印能驱动至少一次增伤、延时结算或掉落收益。 | 不制作独立织梦、飞毯、梦境标签；不把织梦印做成全局通用异常。 |

## 角色组合链路

| 链路 | 标签组合 | P0 价值 |
|-|-|-|
| **塞班 → 奈芙** | 击退 + 缓速 | 先把敌人推离床边，再用月沙区域拖慢回流。 |
| **奈芙 → 影丸** | 缓速 / 标记 + 标记收割 | 先控住关键目标，再由影丸爆发清点。 |
| **棉花 → 影丸** | 标记 / 缓速 + 标记收割 | 棉花用星线标记和梦毯织网稳住关键目标，影丸再追击收割。 |
| **铃音 → 棉花** | 安眠窗口 + 标记 / 缓速 | 主人睡眠度稳定后，切棉花用飞毯绕场铺梦毯织网，延长安全输出窗口。 |
| **铃音 → 塞班** | 安眠 / 护盾 + 击退 | 睡眠压力升高时先救场，再重建前排防线。 |
| **铃音 → 影丸** | 安眠窗口 + 标记收割 | 主人睡眠度稳定后，切影丸处理精英或关键目标。 |

# 6. 组合规则

| 组合 | 效果方向 | 适用场景 |
|-|-|-|
| 击退 + 缓速 | 先把敌人推出床边，再拖慢其重新接近床的速度。 | 塞班冲锋后切奈芙铺流沙，形成床边重置。 |
| 标记 + 缓速 | 让被集火目标难以离开输出区域，提高炮台和范围技能收益。 | 精英怪、Boss 或远程怪需要优先处理时。 |
| 护盾 + 安眠 | 前者保护猫咪或床区，后者稳定主人睡眠，形成防守恢复窗口。 | 主人睡眠度低、猫咪血量低、本能数值又需要处理时。 |
| 标记 + 击退 | 被标记敌人被击退后仍保留集火提示，避免目标丢失。 | Boss 召唤小怪、床边局面混乱时。 |

# 7. UI 与美术口径

P0 的状态 UI 应控制得很轻。普通小怪最多显示一个主状态图标；精英怪和 Boss 可以显示两个。玩家控制猫和主人床区可以显示状态条或光效，但不要把屏幕堆成 MMO 式 buff 列表。状态表现优先用颜色、动作和特效解决，图标只作为补充。

| 标签 | 视觉关键词 | 表现建议 |
|-|-|-|
| 安眠 | 浅蓝、铃声、呼吸光 | 床区或主人睡眠度条周围出现柔和波纹，不做刺眼闪烁。 |
| 缓速 | 沙粒、梦线、飞毯流苏、冰蓝拖尾 | 敌人脚下出现拖拽感，动画步频变慢。 |
| 击退 | 风压、盾撞、冲击线 | 短促冲击波即可，重点是位移反馈清楚。 |
| 标记 | 星线、准星、猫爪印 | Boss 或精英怪头顶/轮廓出现小型标记，不遮挡轮廓。 |
| 护盾 | 银月、圆盾、结界边 | 猫咪或床区外层出现半透明护盾边缘，受击时轻亮。 |

# 8. 三选一强化接口

三选一强化不应只写“伤害 +10%”。每个强化最好至少改变一个状态标签的形态、目标或组合关系。这样肉鸽组合度来自玩家选择，而不是单纯数值堆叠。

| 强化方向 | 标签变化 | 例子 |
|-|-|-|
| 塞班冲锋强化 | 击退变为击退 + 短护盾 | 圆盾冲锋命中敌人后，为塞班生成短护盾，适合高压床边救场。 |
| 奈芙炮台强化 | 缓速变为区域缓速 | 月沙炮台周围形成沙场，敌人进入后持续缓速，适合守路径。 |
| 铃音铃铛强化 | 安眠变为安眠 + 小范围缓速 | 安眠铃周期性稳定睡眠度，同时让靠近床区的敌人变慢。 |
| 棉花织梦强化 | 标记变为标记 + 织梦收益 | 被星线标记的敌人死亡后可额外掉小鱼干，或延长梦毯织网持续时间，服务局内成长。 |

# 9. 梦境事件接口

梦境事件应改变状态标签的环境价值，而不是直接给一堆随机数。当前三个 P0 事件可以这样挂接：手机震动夜提高怪物波峰，使击退和缓速更重要；猫薄荷残香提升输出但加快屎意，使安眠和护盾窗口更重要；闹钟预响让 Boss 前睡眠度持续下降，使安眠成为硬需求。

# 10. P0 开发验收清单

- [ ] 每个一级状态标签都有至少 1 个明确技能来源。

- [ ] 普通敌人能正确响应缓速和击退；Boss 有削弱版处理规则。

- [ ] 主人睡眠度能被安眠类效果影响，并与床防守压力形成闭环。

- [ ] 标记目标在 UI 上可辨识，并至少影响一次输出、索敌或掉落逻辑。

- [ ] 护盾能显示剩余保护状态，且不会与猫咪血量表现混淆。

- [ ] 三选一强化至少有 3 个改变标签形态，而不是只加数值。

- [ ] 梦境事件至少有 1 个能改变某个状态标签的策略价值。
